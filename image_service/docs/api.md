# Image Service API

Базовый префикс всех маршрутов: `/api`.

---

## Общий формат ответа

Большинство эндпоинтов возвращают JSON-обёртку единого вида. Поле `data` содержит полезную нагрузку запроса, `success` — признак успешного выполнения, `err_message` — текст ошибки (пустая строка при успехе), `spread_id` — идентификатор трассировки запроса.

```json
{
  "data": { },
  "success": true,
  "err_message": "",
  "spread_id": "uuid-запроса"
}
```

При ошибке HTTP-статус соответствует типу доменной ошибки, `success` равен `false`, `data` — `null`, а пояснение находится в `err_message`.

**Исключение:** эндпоинты получения бинарного содержимого изображения (`GET /api/guest/image/:id`, `GET /api/admin/image/:id`) возвращают бинарные данные напрямую, без JSON-обёртки.

---

## Аутентификация

Авторизация реализована через JWT и внешний пакет `traineepkg`. При успешном входе (`POST /api/auth/login`) или обновлении токена (`POST /api/auth/refresh`) сервер устанавливает две HttpOnly-cookie:

| Cookie | Назначение |
|--------|------------|
| `access_token` | JWT access-токен для доступа к защищённым эндпоинтам |
| `refresh_token` | JWT refresh-токен для продления сессии |

Срок жизни токенов задаётся переменными окружения `JWT_ACCESS_TTL` и `JWT_REFRESH_TTL`.

Защищённые эндпоинты требуют валидной cookie `access_token`. Для выхода (`POST /api/auth/logout`) обе cookie очищаются.

## Эндпоинты

### Health check

**URL:** `GET /api/health`

Проверяет доступность сервиса. Не требует параметров и аутентификации. Возвращает пустой успешный ответ.

**Входящие параметры:** нет.

**Ответ:** `200 OK`

```json
{
  "data": null,
  "success": true,
  "err_message": "",
  "spread_id": "..."
}
```

---

### Регистрация пользователя

**URL:** `POST /api/auth/register`

Создаёт нового клиента с указанным логином и паролем. Пароль сохраняется в хешированном виде.

**Входящие параметры:**

| Параметр | Расположение | Тип | Обязательный | Описание |
|----------|--------------|-----|--------------|----------|
| `login` | body (JSON) | string | да | Логин пользователя |
| `pass` | body (JSON) | string | да | Пароль |

**Тело запроса:**

```json
{
  "login": "testuser",
  "pass": "123456"
}
```

**Ответ:** `201 Created`

```json
{
  "data": {
    "user": {
      "id": 1,
      "login": "testuser",
      "data": {
        "created_at": "2026-07-03T09:00:00Z"
      }
    }
  },
  "success": true,
  "err_message": "",
  "spread_id": "..."
}
```

### Вход

**URL:** `POST /api/auth/login`

Проверяет логин и пароль, генерирует пару JWT-токенов и устанавливает cookie `access_token` и `refresh_token`. Возвращает данные авторизованного пользователя.

**Входящие параметры:**

| Параметр | Расположение | Тип | Обязательный | Описание |
|----------|--------------|-----|--------------|----------|
| `login` | body (JSON) | string | да | Логин пользователя |
| `pass` | body (JSON) | string | да | Пароль |

**Тело запроса:**

```json
{
  "login": "testuser",
  "pass": "123456"
}
```

**Ответ:** `200 OK`

```json
{
  "data": {
    "user": {
      "id": 1,
      "login": "testuser",
      "data": {
        "created_at": "2026-07-03T09:00:00Z"
      }
    }
  },
  "success": true,
  "err_message": "",
  "spread_id": "..."
}
```

Дополнительно устанавливаются cookie `access_token` и `refresh_token` (HttpOnly, срок жизни — по `JWT_ACCESS_TTL` и `JWT_REFRESH_TTL`).

### Обновление токена

**URL:** `POST /api/auth/refresh`

Проверяет валидность refresh-токена из cookie `refresh_token`, выпускает новую пару JWT-токенов и обновляет cookie. Не требует тела запроса.

**Входящие параметры:**

| Параметр | Расположение | Тип | Обязательный | Описание |
|----------|--------------|-----|--------------|----------|
| `refresh_token` | cookie | string | да | JWT refresh-токен |

**Ответ:** `200 OK`

```json
{
  "data": {
    "user": {
      "id": 1,
      "login": "testuser",
      "data": {
        "created_at": "2026-07-03T09:00:00Z"
      }
    }
  },
  "success": true,
  "err_message": "",
  "spread_id": "..."
}
```

Дополнительно обновляются cookie `access_token` и `refresh_token`.

---

### Выход

**URL:** `POST /api/auth/logout`

Завершает сессию пользователя, очищая cookie `access_token` и `refresh_token`. Не требует тела запроса.

**Входящие параметры:** нет.

**Ответ:** `200 OK`

```json
{
  "data": {
    "message": "logout successful"
  },
  "success": true,
  "err_message": "",
  "spread_id": "..."
}
```

### Загрузка изображения

**URL:** `POST /api/upload`

Сохраняет новое изображение со статусом `unmoderated` и отправляет уведомление сервису модерации. Данные файла передаются в теле запроса в виде base64-строки. Требует валидной cookie `access_token`.

**Входящие параметры:**

| Параметр | Расположение | Тип | Обязательный | Описание |
|----------|--------------|-----|--------------|----------|
| `access_token` | cookie | string | да | JWT access-токен |
| `name` | body (JSON) | string | да | Название изображения |
| `media_type` | body (JSON) | string | да | Тип медиа: `jpeg`, `png`, `gif`, `webp`, `bmp`, `svg`, `tiff`, `ico`, `heic`, `avif` |
| `data` | body (JSON) | string (base64) | да | Содержимое файла в base64 |

**Тело запроса:**

```json
{
  "name": "test_image",
  "media_type": "jpeg",
  "data": "<base64>"
}
```

**Ответ:** `201 Created`

```json
{
  "data": {
    "id": 1,
    "name": "test_image",
    "media_type": "jpeg",
    "status": "unmoderated",
    "created_at": "2026-07-03T09:00:00Z"
  },
  "success": true,
  "err_message": "",
  "spread_id": "..."
}
```

### Получение изображения (гостевой доступ)

**URL:** `GET /api/guest/image/:id`

Возвращает содержимое файла изображения для неавторизованных пользователей. Для изображений со статусом `unmoderated` или `blocked` применяется визуальная обработка (блюр или баннер блокировки). Ответ не оборачивается в общий JSON-формат.

**Входящие параметры:**

| Параметр | Расположение | Тип | Обязательный | Описание |
|----------|--------------|-----|--------------|----------|
| `id` | path | integer | да | Идентификатор изображения |
| `type` | query | string | нет | При значении `icon` возвращается иконка изображения |

**Пример запроса:** `GET /api/guest/image/1?type=icon`

**Ответ:** `200 OK` — тело ответа содержит бинарные данные изображения.

Заголовок `Content-Type` зависит от `media_type` (например, `image/jpeg`, `image/png`, `image/svg+xml`).

### Список немодерированных изображений

**URL:** `GET /api/image/unmoderated`

Возвращает страницу изображений со статусом `unmoderated`. Требует валидной cookie `access_token` и поддерживает пагинацию через query-параметры.

**Входящие параметры:**

| Параметр | Расположение | Тип | Обязательный | Описание |
|----------|--------------|-----|--------------|----------|
| `access_token` | cookie | string | да | JWT access-токен |
| `page` | query | integer | нет | Номер страницы (с нуля). По умолчанию — значение из конфига (`PAGINATION_DEFAULT_PAGE`, по умолчанию `0`) |
| `page_size` | query | integer | нет | Размер страницы. По умолчанию — значение из конфига (`PAGINATION_DEFAULT_PAGE_SIZE`, по умолчанию `10`) |

**Пример запроса:** `GET /api/image/unmoderated?page=0&page_size=2`

**Ответ:** `200 OK`

```json
{
  "data": {
    "images": [
      {
        "id": 1,
        "name": "test_image",
        "media_type": "jpeg",
        "crated_at": "2026-07-03T09:00:00Z",
        "status": "unmoderated"
      }
    ],
    "count": 1,
    "total_count": 1
  },
  "success": true,
  "err_message": "",
  "spread_id": "..."
}
```

## Возможные ошибки

Все ошибки (кроме эндпоинтов получения бинарного содержимого изображения при отсутствии файла) возвращаются в общем JSON-формате с `success: false`.

| HTTP-статус | Тип ошибки | Когда возникает | Пример `err_message` |
|-------------|------------|-----------------|----------------------|
| `400 Bad Request` | `INVALID_PARAMETERS` | Невалидное тело запроса, параметры или id | `неверный параметр body со значением <nil>: invalid request body` |
| `400 Bad Request` | `INVALID_PARAMETERS` | Пустые или неподдерживаемые поля изображения | `неверный параметр media_type со значением bmp: unsupported media type` |
| `400 Bad Request` | `INVALID_PARAMETERS` | Некорректная пагинация | `неверный параметр page_size со значением -1: page_size must be positive` |
| `400 Bad Request` | `INVALID_PARAMETERS` | Нечисловой `id` в path | `неверный параметр id со значением abc: invalid image id` |
| `401 Unauthorized` | `UNAUTHORIZED` | Неверный логин или пароль | `неавторизованный доступ: invalid credentials` |
| `401 Unauthorized` | `TOKEN` | Отсутствует, просрочен или невалиден JWT-токен | `Ошибка обработки токена Токен не найден` |
| `404 Not Found` | `NOT_FOUND` | Изображение или пользователь не найдены | `image с ID 99 не найден` |
| `409 Conflict` | `UNIQUE_CONSTRAINT` | Пользователь с таким логином уже существует | `нарушение уникальности для client: поле name со значением testuser уже существует` |
| `500 Internal Server Error` | `INTERNAL_SERVICE` | Сбой БД, файлового хранилища или другая внутренняя ошибка | `внутренняя ошибка сервиса` |

### Статусы изображений

| Значение | Описание |
|----------|----------|
| `unmoderated` | Ожидает модерации |
| `approved` | Одобрено |
| `blocked` | Заблокировано |
