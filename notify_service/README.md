# Сервис оповещений

Микросервис для рассылки оповещений пользователям в Telegram, Email и на различные Webhook'и.

## Запуск

```bash
make run
```

В результате запуска команды докер начнет сборку проекта. По завершению должно быть написано в каждом из docker image **started**

## Переменные окружения

1. **DATABASE_CONNECT** - адрес подключения к базе данных
2. **BOT_TOKEN** - токен Telegram бота
3. **smtp...**
    - **smtpHost** - хост для SMTP
    - **smtpPort** - порт для SMTP
    - **smtpEmail** - адрес почты для SMTP
    - **smtpPassword** - пароль для входа в почту для SMTP
4. **WORKER_COUNT** - максимальное количество одновременно работающих горутин
5. **JWT_SECRET** - секрет для JWT

## Эндпоинты

- **POST api/notify** - по этому эндпоинту отправляются уведомления пользователям
- **POST api/notify_types** - по этому эндпоинту сохраняются в базу настртойки, которые были выставлены в админ-панели
- **GET api/get_notify_settings** - по этому эндпоинту получаются настройки из базы и переносятся в админ-панель
- **POST api/moderator_login** - по этому эндпоинту реализована логика входа в админ-панель

## Структура запросов к эндпоинтам

```json
{
    "notify_type": "тип_уведомления",
    "telegram_id": айди_юзера_в_тг (int),
    "email": "адрес почты юзера",
    "message": "сообщение для уведомления"
}
```
